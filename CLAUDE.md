# CLAUDE.md

## 公众号文章写作指南

**核心定位：给用户一个借口。用户不是来找答案的，是来找"不是我的错"的。**

先信任人，再信任文章。先给借口，再给方法。

### 写作流程

1. 读取本文件 → 2. 查重避雷（读 `articles/` 目录）→ 3. 选题（选"借口"最深的主题）→ 4. 撰写正文 → 5. 生成标题

### 核心原则

- **给借口优于给答案**：每个痛点的核心不是教解决方法，是告诉用户"不是你的问题，是XX害的"
- 用词温和：可以、可能、有助于，不用绝对化词语
- 强调个体差异，给用户希望而非承诺
- 每篇文章只讲一个方法

### 标题规则

**固定格式：你真美 | XXXX内容**

**信任标题公式（优先级从高到低）：**
1. **悬念缺口型**："你以为是A，其实可能是B"（不直接给答案）
2. **借口型**："不是你的错，是XX害的"
3. **痛点设疑型**："为什么你的XX总是XX？"（问而不答）

**禁忌：**
- ❌ 标题里直接暴露答案（如"睡觉去黑眼圈"→答案已暴露，用户无点击欲望）
- ❌ 教科书式标题（如"XX的方法""XX的技巧"）

### 开头规则（决定用户留不留）

**开头必须短（3-5句内），必须软，必须给借口。**

**信任开头公式：**
1. 一句短判断（否定用户的自我怀疑）
2. 一句共情（你其实不是不想/不会）
3. 一句归因转移（不是你错了，是XX错了）

**示例：**
```
你其实不是不想拍。
你只是怕。怕看到拍完之后的自己。
不是。是镜头从一开始就在歪曲你。
```

**禁忌：**
- ❌ "你是不是也有XX困扰？"（审判式开头）
- ❌ "很多人都有这个烦恼"（太泛，没有刺痛感）
- ❌ 长篇铺垫，超过5句还没说到重点
- ❌ "昨天后台有个读者问我"（虚构读者提问）
- ❌ "最近收到很多粉丝留言"（虚假互动）

### 信任建立

**用户先信任作者，才信任文章。**

**同路人视角 > 专家视角：**
- ✅ "有一段时间我也这样"、"后来才发现"
- ❌ "研究表明"、"大多数人的错误做法是"（居高临下）

**失败经历必须具体，必须不包装：**
- ✅ "我用了整整一年撕拉鼻贴，用到鼻子上的毛孔肉眼可见地变大了"
- ❌ "我以前也用撕拉鼻贴，后来发现不好就停了"

**传播驱动力来源：**
读者转发不是因为文章有用，是因为**"这篇写的就是我"**——转发是在表达自己。

### 四大写作手法（一篇只重点用一个）

1. **痛点破局法**：标题即痛点，开头即共鸣
2. **情绪共鸣法**：先共情，后指导，给借口
3. **认知减负法**：模块化排版，大白话表达
4. **价值锚定法**：正反对比，指令式实操

### 文章结构（4-6个 `##` 章节）

1. **引入**（短、软、给借口）：否定自我怀疑，转移归因
2. **真相/依据**（200-400字）：用比喻讲科学，不用论文语气
3. **方法**（200-400字）：3步够了，降门槛
4. **个人经历**（150-200字）：具体失败经历，不包装
5. **效果/鼓励**（100-150字）：降门槛，温和结语

**结尾必须提醒：点赞、分享、点小爱心 ❤️**

### 🧠 脑科学驱动排版优化 (SPA 策略)

1. **封面级大标题**：必须使用 `# 类别 | 主标题` 格式
2. **三级锚定点**：
   - **H2 章节点**：用于切换话题
   - **数字气泡 (1. 2. 3.)**：用于步骤或关键点
   - **呼吸分割线 (---)**：每章节结尾必用
3. **碎步跑动**：单段文字控制在 **3行以内**，句子短小

### 模板语法

| 语法 | 用途 |
|------|------|
| `# 类别 \| 标题` | 封面级大标题 |
| `## 章节标题` | 主要章节（4-6个） |
| `### 氛围行` | emoji 氛围或次级引导 |
| `#### 小标题\|大标题` | 重点方法入口（核心锚点） |
| `---` | 呼吸分割线 |
| `> 说明卡` | 结论、提醒、核心解释 |
| `!! 高亮块` | 关键点金句（1-2次） |
| `` `关键词` `` | 视觉检索 Tag |
| 无序列表 | 步骤、提醒、注意事项 |

**准则：**
1. **适合最重要**：不需要堆砌所有格式。选2-3种模块即可
2. `!!` 每篇最多1-2次
3. 段落短小，单段不超过3行
