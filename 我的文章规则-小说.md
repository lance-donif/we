# 阿柒故事屋 · 短篇故事规则 V3.0

```text
项目类型：software/content-system
项目描述：为公众号女频/大众情绪读者生产高点击、高追读、高书架转化的口述体短篇故事
成功标准：点击率高于 6%；加入书架率提升；6 篇里至少 2 篇有持续追读和评论
关键文件：优先修改本规则；不覆盖已发布小说原文；测试数据记录到 memory.md
```

## 一、主要矛盾

当前数据：展现 2130，阅读 100，点击率 4.69%，点赞 13，评论 1，加入书架 2。

主要矛盾不是没推荐，而是点进来后没有追读冲动。

旧规则写出来像“小说大纲展开”；热门故事像“当事人亲口讲旧事”。旧规则重设定完整，热门故事重前三屏生死。旧规则让读者看故事，热门故事让读者立刻站队。

短篇不是标签拼盘，也不是写作课范文。短篇是两分钟内让读者说：我倒要看看她怎么翻盘。

## 二、读者根据地

- 读者根据地：中午碎片时间、晚上睡前、通勤刷文。
- 目标读者：想看解气、翻盘、后悔、重生、换命、婆媳、婚姻清算。
- 阅读状态：手指随时准备划走，所以每三五行就要给一个继续看的理由。

群众喜欢的不是“设定多”，而是“第一句就知道谁欺负谁”。

## 三、标题

标题要像热榜故事，不要像文学作品。

```text
关系压迫 + 极端处境 + 反常结果
```

可用方向：

- 怀二胎去产检时，丈夫情人也在门诊
- 重生后我还是嫁给了前夫
- 娘娘千岁，不如高考万岁
- 我流产那天，婆婆把我赶出家门
- 我死后三年，丈夫终于看见了我留下的账本

标题要有关系词：丈夫、婆婆、前夫、妹妹、闺蜜、女儿、继母、未婚夫、娘娘。

标题要有命运词：重生、流产、产检、离婚、替身、换命、背叛、赶出家门、回到那天。

禁止标题：女总裁逆袭短篇、她被看不起后翻盘、一场关于尊严的胜利。这些像作者总结，不像读者会点的故事。

## 四、格式

正文不要放标题。微信标题有单独输入框。

必须用第一人称口述体。像当事人在手机屏幕前平静地讲一件旧事。

- 每段 1 到 3 句。
- 多空行。
- 不铺背景。
- 不解释世界观。
- 先给命运问题或当场羞辱。

分节不用“第一章、第二章”。用数字分节：

```text
1
```

每节 800 到 1500 字。数字前后空行。读者在手机上不需要章名，只需要换气口。

段落密度：

- 前 1000 字不能出现 5 行以上大段。
- 全文每 200 到 300 字必须有一句台词、一个动作或一个新信息。
- 关键爽点单独成段。

## 五、口吻

- 冷静，不哭喊。
- 句子短，不端着。
- 情绪狠，但不解释狠。
- 反击前越平静，反击时越爽。

多用：我点点头、我没说话、我看着他、我终于笑了、后来我才知道。

少用：我内心五味杂陈、命运的齿轮开始转动、这是一场关于尊严和爱情的较量。

反打句必须接住对方原话里的具体词，先处理眼前物件或动作，再引出主线。禁止为了点题硬转“科举、前程、命运”这类抽象词。

## 六、前三屏

开局800字必须完成：

1. 主角被伤到哪一步。
2. 伤她的人是谁。
3. 这段关系为什么扎心。
4. 主角会走向哪种反转。

开头最好用四连击：

```text
一句命运问题
一句冷静回答
一句关系暴击
一句接住原话的反常动作
```

```text
如果再见到害死你的人，你会不会扑上去撕了她？

我不会。

我会把她亲手送上花轿。

因为只有那样，她才会替我走完上辈子的死路。
```

禁止开头：天气、房间、公司背景、世界观说明、主角履历、长篇回忆。

## 七、人物和推进

一篇只打一个核心情绪。

主角占“冤”：被背叛、被轻贱、被替代、被误会、被抢命、被逼着忍。

反派占“狠”：必须做具体坏事，不要只写“她很坏”。可写当众羞辱、抢孩子、抢婚约、逼签字、赶出家门、在主角最惨时说风凉话、把主角功劳给别人。

关键配角占“晚”：丈夫、前夫、太子、少爷可以后悔，但不能太早清白。他必须先伤到人，读者才有“你也有今天”的快感。

每节只推进一个战果：

1. 受辱：让读者站到主角这边。
2. 忍住：主角不立刻哭闹，留下悬念。
3. 反常：主角做一个不符合弱者身份的动作。
4. 反击：证据、身份、记忆、制度、命运开始转向。
5. 后悔：伤害过她的人开始慌。
6. 清算：主角赢，但不回头。

每 1500 字至少一次情绪波动：更惨、更气、更狠、更接近真相。

## 八、章末钩子和结尾

章末钩子不讲道理，只丢下一把刀。

钩子类型：手机弹出旧消息；门外站着不该出现的人；主角说出反常话；证据只露半截；反派以为赢了，其实踩进坑里。

1. 爽结尾：反派受报应，主角赢。
2. 刀后回甘：前面虐，最后尊严回来。
3. 反转结尾：最后一击揭穿更大真相。

可以留“下一章”的故事内悬念，但不要在正文里写“加入书架”“求关注”“读者”等平台话术。好的结尾不是告诉读者去关注，而是让读者自己舍不得走。

## 九、自检

- [ ] 标题是否像热门故事，而不是作者总结。
- [ ] 正文是否没有放标题。
- [ ] 是否使用第一人称口述体。
- [ ] 是否用数字分节，而不是“第几章”章名。
- [ ] 前 300 字是否有短段、留白、命运问题或当场羞辱。
- [ ] 开局800字是否立住委屈、压迫、关系、反转承诺。
- [ ] 是否一篇只打一个核心情绪。
- [ ] 每节是否只推进一个战果。
- [ ] 每 1500 字是否至少一次情绪波动。
- [ ] 结尾是否没有平台话术，但有继续看的理由。

## 十、低浏览诊断闸门

低浏览先看数据，不先改口吻。

每篇发布 24 小时后先分诊：

1. 展现很少：这是题材池/账号标签/发布入口问题，暂停改正文。
2. 展现不少但点击率低于 4%：才改标题、封面和开头第一屏。
3. 点击率不低但书架、评论、追读弱：才改爽点、钩子和结尾。

若连续 3 篇展现很少，下一轮只测一个根据地：婚姻背叛或婆媳羞辱，不混古代科举、商战、修仙、男频脑洞。

## 十一、发布测试

不要再一口气写十几本同质短篇。按变量测试。

第一轮：测开头。同类题材，分别用“命运问题”“当场羞辱”“重生醒来”三种开头，比前三屏留人。

第二轮：测格式。同一题材，一版章名结构，一版数字分节口述体，看完读率和加入书架率。

第三轮：测题材根据地。婚姻背叛、婆媳羞辱、重生换命、古代女主读书翻身分别打小样，看哪条线评论和书架更高。

记录到 `memory.md`：

```markdown
### [日期/轮次] [小说标题]
- 问题：这次验证什么变量
- 方案：开头、口吻、格式、钩子做了什么调整
- 结果：展现 / 阅读 / 点击率 / 评论 / 点赞 / 加入书架
- 教训：读者到底在哪一段走了
```

最后一句：群众已经投票了。短篇要先像一个真实的人在讲旧事，再像一把刀慢慢亮出来。
